Introduction¶

Les services de streaming ont examiné les moyens de catégoriser la musique pour permettre des recommandations personnalisées. Aujourd’hui, nous examinerons les données compilées par un groupe de recherche connu sous le nom de The Echo Nest. Notre objectif est de parcourir cet ensemble de données et de classer les chansons comme étant soit « Hip-Hop » ou « Rock » - le tout sans en écouter une seule nous-mêmes.

1. Preparer notre dataset¶

Nettoyer nos données, effectuer une visualisation exploratoire des données et à utiliser la réduction des fonctionnalités dans le but d’alimenter nos données grâce à des algorithmes d’apprentissage automatique simples, tels que des arbres de décision et une régression logistique.

3. Normalisation des données¶

Comme nous n’avons pas trouvé de corrélations particulièrement fortes entre nos caractéristiques, nous pouvons plutôt utiliser une approche commune pour réduire le nombre de caractéristiques appelée analyse en composantes principales (APC).

L’APC fait pivoter les données le long de l’axe de variance la plus élevée, ce qui nous permet de déterminer la contribution relative de chaque caractéristique de nos données à la variance entre les classes.

Malheureusement, il ne semble pas y avoir de coude clair dans ce diagramme, ce qui signifie qu’il n’est pas simple de trouver le nombre de dimensions intrinsèques en utilisant cette méthode.

Nous pouvons également examiner le diagramme de variance expliquée cumulative pour déterminer combien de caractéristiques sont nécessaires pour expliquer, disons, environ 85% de la variance (les seuils sont quelque peu arbitraires ici, et généralement décidés par des « règles empiriques »). Une fois que nous avons déterminé le nombre approprié de composants, nous pouvons effectuer un PCA avec autant de composants, réduisant idéalement la dimensionnalité de nos données.

In [9]:
# Import numpy
import numpy as np

# Calculate the cumulative explained variance
cum_exp_variance = np.cumsum(exp_variance)

# Plot the cumulative explained variance and draw a dashed line at 0.85.
fig, ax = plt.subplots()
ax.plot(cum_exp_variance)
ax.axhline(y=0.85, linestyle='--')

# choose the n_components where about 85% of our variance can be explained
n_components = 6

# Perform PCA with the chosen number of components and project data onto components
pca = PCA(n_components, random_state=10)
pca.fit(scaled_train_features)
pca_projection = pca.transform(scaled_train_features)

Maintenant, nous pouvons utiliser la projection PCA de dimension inférieure des données pour classer les chansons en genres. Pour ce faire, nous devons d’abord diviser notre ensemble de données en sous-ensembles « train » et « test », où le sous-ensemble « train » sera utilisé pour entraîner notre modèle tandis que le jeu de données « test » permet la validation des performances du modèle.

6. Arbre de décision pour classer le genre des chansons¶

Ici, nous utiliserons un algorithme simple connu sous le nom d’arbre de décision. Les arbres de décision sont des classificateurs basés sur des règles qui prennent en compte les caractéristiques et suivent une « arborescence » de décisions binaires pour finalement classer un point de données dans l’une des deux catégories ou plus. En plus d’être faciles à utiliser et à interpréter, les arbres de décision nous permettent de visualiser l’organigramme logique que le modèle génère à partir des données d’entraînement.

8. Équilibrez nos données pour de meilleures performances¶

Nos deux modèles font de même, avec une précision moyenne de 87% chacun. Cependant, en regardant notre rapport de classification, nous pouvons voir que les chansons rock sont assez bien classées, mais les chansons hip-hop sont classées de manière disproportionnée comme des chansons rock.

L’équilibrage de nos données a éliminé les biais en faveur de la classe la plus répandue. Pour avoir une bonne idée de la performance de nos modèles, nous pouvons appliquer ce qu’on appelle la validation croisée (CV). Cette étape nous permet de comparer les modèles de manière plus rigoureuse.

La validation croisée tente de diviser les données de plusieurs manières et de tester le modèle sur chacune des divisions. Bien qu’il existe de nombreuses méthodes de CV différentes, toutes avec leurs propres avantages et inconvénients, nous utiliserons ce que l’on appelle le CV K-fold ici. K-fold divise d’abord les données en K sous-ensembles différents et de taille égale.

from sklearn.model_selection import KFold, cross_val_score

# Set up our K-fold cross-validation
kf = KFold(10)

tree = DecisionTreeClassifier(random_state=10)
logreg = LogisticRegression(random_state=10)

# Train our models using KFold cv
tree_score = cross_val_score(tree, pca_projection, labels, cv=kf)
logit_score = cross_val_score(logreg, pca_projection, labels, cv=kf)

# Print the mean of each array o scores
print("Decision Tree:", np.mean(tree_score), "Logistic Regression:", np.mean(logit_score))
